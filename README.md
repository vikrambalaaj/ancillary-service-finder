# Ancillary Service Finder

A Node.js + TypeScript app that collects a Farelogix **ServiceListRQ** (Emirates EK cabin-upgrade ancillary) request as input and returns the matching **ServiceListRS** response as output.

## What it does

- **Form-driven input** with full validation, mandatory-field enforcement, and input-type-aware widgets:
  - `date` picker for dates, `time` picker for times, `number` for flight number, `password` for credentials, dropdowns (`select`) for airports, countries, airlines, channels, cabins, class of service, equipment, and Y/N flags.
- **Search** button validates on both client and server, builds the SOAP request XML, and returns the parsed response (services, pricing, taxes, rules, booking instructions).
- **Demo toggle** (top-right). When ON it auto-fills every field from a known-good sample and submits immediately — no prompts. When OFF it clears the form and requires you to fill in all mandatory fields before Search will work.
- Response panel has three tabs: **Services** (rich cards), **Response XML** (raw), **Request XML** (the generated SOAP envelope).

## Stack

- Backend: Node.js + Express + TypeScript (`src/`)
- Frontend: vanilla TypeScript bundled by esbuild (`public/`)
- XML: `fast-xml-parser` for response parsing; hand-rolled typed builder for the request envelope
- No Python anywhere

## Project layout

```

  src/

    routes/

```

## Run

```bash
cd ancillary-app
npm install

# Dev (hot-reload server + watch/rebuild frontend)
npm run dev
#  -> server on http://localhost:3000, esbuild bundles public/app.ts

# Production build + start
npm run build
npm start
```

Then open http://localhost:3000.

- Flip the **Demo** toggle (top-right) to auto-fill the form, then click **Search** to call the live endpoint.
- Turn Demo OFF to fill the form yourself; mandatory fields are marked `*` and validated on Search.

## Endpoint configuration (env)

The endpoint URL is read from environment variables (loaded from `.env` automatically via Node's `process.loadEnvFile`). Defaults are in `.env`:

| Variable | Default | Purpose |
| --- | --- | --- |
| `FARELOGIX_URL` | `https://ekd.farelogix.com/sandbox-uat/flxm` | Farelogix ServiceList endpoint hit by Search |
| `FARELOGIX_TIMEOUT_MS` | `20000` | Request timeout |
| `USE_SAMPLE_RESPONSE` | `false` | Set to `true` to skip the live call and return the attached sample (offline mode) |
| `ALWAYS_SHOW_SERVICES` | `true` | When the live endpoint returns no services, fall back to sample services so the Services tab is never empty |
| `CORS_ORIGIN` | _(all)_ | Comma-separated allowlist of origins for `/api`; unset allows all origins |
| `PORT` | `3000` | Server port |

The API enables CORS for all origins by default (preflight `OPTIONS` handled), so the frontend can call `/api/*` from any origin. Restrict it with `CORS_ORIGIN` when deploying. The browser never calls Farelogix directly — the Node server proxies the SOAP request, which avoids the cross-origin block on the Farelogix endpoint.

If the live call fails (network/timeout/SOAP fault), Search returns HTTP 502 with a descriptive message and the endpoint it tried.
